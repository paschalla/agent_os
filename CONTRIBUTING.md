## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report.

- **Use a clear and descriptive title**.
- **Describe the steps to reproduce**.
- **Include screenshots** if the issue is visual (like the frontend).

### Suggesting Enhancements

- **Explain why this enhancement would be useful**.
- **Provide context** on how it fits into the current architecture (Turtle Shell, CLI, etc.).

### Pull Requests

1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure your code lints and follows the existing style (Python PEP8, standard JS/React patterns).
4.  Issue that pull request!

## Styleguides

### Python

- We follow PEP 8.
- Use type hints (`def func(a: int) -> str:`) wherever possible.

### React/Frontend

- Use functional components and hooks.
- Keep components small and reusable.
